## 2.2.1 - Exams Creation Validation

**Fecha**: 27 Enero 2026  
**Severidad**: 🔴 Critical  
**Módulo**: Exams

### Síntoma

Botón "Crear Examen" no funcionaba. No había feedback visual.

### Causa Raíz

Validación de Zod fallaba silenciosamente por el campo `number` (opcional):

```typescript
// ❌ ANTES
number: z.number().optional()
// Problema: input vacío = string "", no undefined
```

### Solución

Transform en Zod + setValueAs en form:

```typescript
// Schema
number: z.number().optional().or(z.literal('').transform(() => undefined))

// Form
{...register('number', {
  setValueAs: (v) => (v === '' ? undefined : parseInt(v, 10))
})}
```

## 2.3.1 - Deletion Requires Reload

**Fecha**: 27 Enero 2026  
**Severidad**: 🟠 High  
**Módulo**: Exams (UX)

### Síntoma

Al eliminar un examen, el Server Action funcionaba pero la UI no se actualizaba. Requería F5 manual.

### Causa Raíz

Faltaba callback `onDelete` en la cadena de componentes:

```
Page → ExamList → ExamCard
```

ExamCard eliminaba pero no notificaba al Page para recargar datos.

### Solución

Agregar callbacks en cadena:

```typescript
// Page
const handleDelete = () => loadData();

// ExamList
<ExamCard onDelete={() => onDelete(exam.id)} />

// ExamCard (después del delete)
if (!result.error) {
  onDelete(); // Notificar al padre
}
```

### Lecciones Aprendidas

1. React unidirectional data flow requiere callbacks explícitos
2. Aplicar pattern consistente en todos los CRUDs
