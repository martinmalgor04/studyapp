## 1.1.1 - PostgreSQL Trigger Error (users.name)

**Fecha**: 26 Enero 2026  
**Severidad**: 🔴 Critical  
**Módulo**: Database / Auth

### Síntoma

Error durante signup:
```
ERROR: column "name" of relation "users" does not exist
```

### Causa Raíz

El trigger `handle_new_user()` intentaba acceder a `auth.users.name` que no existe. Los metadatos del usuario están en `raw_user_meta_data`.

### Solución

Modificar trigger para acceder correctamente:

```sql
-- ✅ CORRECTO
INSERT INTO public.users (id, email, name)
VALUES (
  NEW.id,
  NEW.email,
  NEW.raw_user_meta_data->>'name'
);
```

Agregado `SET search_path = public` para evitar ambigüedad.

### Archivos Modificados

- `supabase/migrations/20240126000001_initial_schema.sql`

### Impacto

- Bloqueaba signup completamente
- Requirió `npx supabase db reset`

## 1.2.1 - Module Resolution (@/lib)

**Fecha**: 26 Enero 2026  
**Severidad**: 🔴 Critical  
**Módulo**: Build / TypeScript

### Síntoma

```
Module not found: Can't resolve '@/lib/supabase/middleware'
```

### Solución

Agregar `paths` en `tsconfig.json`:

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

## 1.1.2 - Docker/Podman Compatibility

**Fecha**: 26 Enero 2026  
**Severidad**: 🟠 High  
**Módulo**: DevOps

### Síntoma

Supabase CLI requiere Docker pero usuario tenía Podman.

### Solución

Cambiar de Podman a Docker después de varios intentos con `DOCKER_HOST`.

### Lecciones Aprendidas

1. Supabase CLI no soporta Podman oficialmente
2. Docker es más estándar para desarrollo local

## 1.3.1 - Supabase CLI Hanging

**Fecha**: 26 Enero 2026  
**Severidad**: 🔵 Low  
**Módulo**: DevOps

### Síntoma

`npx supabase start` se colgaba indefinidamente.

### Solución

```bash

```

Restart completo resolvió glitch de networking.
